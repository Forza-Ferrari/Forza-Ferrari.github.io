题意：给一些图论算法构造数据，每次放过一个卡掉一个，定义“卡掉”的标准是代码中的计数器自加超过 10610^6106 次。只有你构造的数据中整数的个数小于一个给定值 TTT 时才能获得满分。

subtask 1(3pts)\mathrm{subtask\ 1(3pts)}subtask 1(3pts)

subtask 2(7pts)\mathrm{subtask\ 2(7pts)}subtask 2(7pts)

放过 FloydFloydFloyd 卡掉 Bellman−FordBellman-FordBellman−Ford，T=2222T=2222T=2222。

根据 sub1sub1sub1 得出的结论，我们的点数不能多于 100100100。观察 Bellman−FordBellman-FordBellman−Ford 的实现，发现计数器每跑一轮就会增加 ∑ni\sum n_i∑ni​，由于代码在一轮下来没有松弛成功的情况下就会推出，所以我们要让它跑满 V−1V-1V−1 轮。

考虑 Bellman−FordBellman-FordBellman−Ford 其实是一个顺推的过程，比如如果图是一条 0→V−10\rightarrow V-10→V−1 的有向链，其实一轮就已经完成了松弛。

那么我们反过来，把图建成一条 V−1→0V-1\rightarrow0V−1→0 的有向链，这样每次松弛会恰好成功一次，每次询问的时候起点设成 V−1V-1V−1 就可以卡满。

询问尽量多，Q=10Q=10Q=10，这样会进行 Q(V−1)∑ni=98010Q(V-1)\sum n_i=98010Q(V−1)∑ni​=98010 次运算，远远不到 10610^6106。

其实这样构造数据中的整数只有 320320320 个，浪费了很多资源。注意到其实题目没有要求是简单图，我们大可以把多出来的整数变成重边塞进去。

算一下 2222−22−100100×2=10.5\frac{2222-22-100}{100\times 2}=10.5100×22222−22−100​=10.5，每条边都可以变成十条重边。这样 Q(V−1)∑ni=980100Q(V-1)\sum n_i=980100Q(V−1)∑ni​=980100，还差一点。

其实还是有点浪费，我们考虑利用刚才扔掉的 0.50.50.5 条边。其实我们只要每两个点连出去 212121 条边就是合法的，那么我们直接分奇偶连 101010 和 111111 条重边，这样 Q(V−1)∑ni=1039500Q(V-1)\sum n_i=1039500Q(V−1)∑ni​=1039500，刚刚好。

#include<iostream>
#include<cstdio>
using namespace std;
int n,q;
int main()
{
    freopen("2.txt","w",stdout);
    n=100;
    cout<<n<<endl<<11<<endl;
    for(int i=1;i<=11;++i)
        cout<<n-1<<" 1 ";
    cout<<endl;
    for(int i=1;i<n;++i)
    {
        if(i&1)
        {
            cout<<"10 ";
            for(int j=1;j<=10;++j)
                cout<<i-1<<" 1 ";
        }
        else
        {
            cout<<"11 ";
            for(int j=1;j<=11;++j)
                cout<<i-1<<" 1 ";
        }
        cout<<endl;
    }
    q=10;
    cout<<q<<endl;
    while(q--)
        cout<<n-1<<" 0"<<endl;
    return 0;
}

subtask 4(17pts)\mathrm{subtask\ 4(17pts)}subtask 4(17pts)

放过 FloydFloydFloyd 卡掉 DijkstraDijkstraDijkstra，T=157T=157T=157。

这个有点难度。

随了很多组数据，发现这个 DijkstraDijkstraDijkstra 表现出色且稳定，数据规模还限制得这么小，真的能卡掉吗？

发现题目中限制的是边权的绝对值，也就是说可以有负权边，众所周知 DijkstraDijkstraDijkstra 在有负权边的时候是可以卡成指数级别的，我们可以在这个上面做文章。

构造这种数据的核心在于造一个长链出来，先用小的正边权把 DijkstraDijkstraDijkstra 骗到链尾，然后外挂很多点，点的入边是一个大的正权边，出边是一个绝对值比入边大的负权边，这样最短路实际上是走外挂点的那条，但是 DijkstraDijkstraDijkstra 已经被链骗过去更新了一遍整个图了，所以它会重新松弛，多重复几次就变成指数级了，这样算一下其实 T=157T=157T=157 还很松，这里构造的数据 V=33,Q=10V=33,Q=10V=33,Q=10。

#include<iostream>
#include<cstdio>
using namespace std;
int n,q;
int main()
{
    freopen("4.txt","w",stdout);
    n=33;
    cout<<n<<endl;
    for(int i=2;i<n;i+=2)
    {
        cout<<"2 "<<i-1<<" "<<(1<<(17-i/2-1))<<" "<<i<<" 0"<<endl;
        cout<<"1 "<<i<<" "<<-(1<<(17-i/2))<<" "<<endl;
    }
    cout<<0<<endl;
    q=10;
    cout<<q<<endl;
    while(q--)
        cout<<0<<" "<<n-1<<endl;

}

#include<iostream>

subtask 8(25pts)\mathrm{subtask\ 8(25pts)}subtask 8(25pts)

染色，放过暴力，让你卡掉的算法是钦定过不了的。

道理我都懂，这一下子送 252525 分是？

怎么放过这个暴力？把颜色数造得非常小即可。那我们点数开到最大 V=999V=999V=999，然后先来连成一个环，然后其实就是人类智慧，只要把边凑够，没有度数特别大的点就好了。我的做法是再来一轮，每次 i−1i-1i−1 和 2i−12i-12i−1 连边，这样最后还需要连 555 条才能到 150115011501 直接首尾相连即可。这样算一下一共 2×1501+2=30042\times 1501+2=30042×1501+2=3004，刚刚好。

#include<iostream>
#include<cstdio>
using namespace std;
int n,m;
int main()
{
    freopen("8.txt","w",stdout);
    n=999,m=1501;
    cout<<n<<" "<<m<<endl;
    for(int i=1;i<n;++i)
        cout<<i-1<<" "<<i<<endl;
    for(int i=2;i<<1<=n;++i)
        cout<<i-1<<" "<<(i<<1)-1<<endl;
    for(int i=0;i<5;++i)
        cout<<i<<" "<<n-1-i<<endl;
    return 0;
}
